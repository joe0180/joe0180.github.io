Networking Concepts Demonstrated

DNS

The domain is resolved to an IP address using DNS. When a user enters the domain, the request is sent to a recursive DNS resolver, which queries the authoritative GitHub Pages name servers. This returns A (IPv4) or AAAA (IPv6) records containing the server IP.

IP Addressing

This website supports both IPv4 and IPv6, demonstrating dual-stack networking. This ensures compatibility across different devices and networks.

Protocols

The website uses HTTPS for secure communication. HTTP requests are automatically redirected (status 307) to HTTPS, enforcing encryption. A successful request returns a "200 OK" response after the secure connection is established.

Security

XSS protection is demonstrated using DOMPurify to sanitize unsafe input. HTTPS ensures encrypted communication between client and server.

Summary

This project demonstrates real-world networking concepts including DNS resolution, IP addressing, and secure communication using HTTPS.
